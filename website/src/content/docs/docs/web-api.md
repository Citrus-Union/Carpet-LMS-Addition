## Base

- Service address: `http://<server-ip>:<port>`
- Port is from `checkStorageConfig.json` field `port`.

## Auth

- Header: `Authorization: Bearer <token>`
- If `noPassword: true`, `/api/storage/getData` can be called without token.

## POST /api/login

- Request body:

```json
{
  "username": "admin",
  "password": "your-password"
}
```

- Success response:

```json
{
  "username": "admin",
  "token": "jwt-or-token-string"
}
```

## GET /api/storage/getData

- Method: `GET`
- Response uses compact keys.
- Dimension key mapping:
- `"0"`: overworld
- `"-1"`: nether
- `"1"`: end

Example response:

```json
[
  {
    "n": "main",
    "d": {
      "i": {
        "minecraft:barrel": {
          "c": 1152,
          "0": [
            [1, 100, 0, 576],
            [-1, 100, 0, 576]
          ],
          "-1": [],
          "1": []
        }
      },
      "e": {
        "0": [[0, 100, 0]],
        "-1": [],
        "1": []
      }
    }
  }
]
```

Field notes:

- `n`: storage name.
- `d`: storage data.
- `d.i`: item map keyed by `itemId`.
- `d.i[itemId].c`: total item count.
- `d.i[itemId]["0" | "-1" | "1"]`: position list in one dimension, each entry is `[x, y, z, n]`.
- `d.e["0" | "-1" | "1"]`: error position list in one dimension, each entry is `[x, y, z]`.

## POST /api/storage/getItem

- Method: `POST`
- Request body:

```json
{
  "i": "minecraft:stone",
  "c": 64
}
```

Field notes:

- `i`: item id.
- `c`: requested count, must be integer >= 1.

Success response:

```json
[
  {
    "n": "bot_getitem_1",
    "i": "minecraft:stone",
    "c": 64
  }
]
```

Field notes:

- `n`: bot name.
- `i`: item id.
- `c`: count fetched by this bot.
- Commands and log lines are generated by frontend from `n/i/c`.

## Error response

Common error shape:

```json
{
  "status": "400",
  "message": "error message"
}
```
